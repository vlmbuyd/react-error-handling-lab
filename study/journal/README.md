# 깨달음 일지 (journal)

`/wrap`이 세션 끝에 `YYYY-MM-DD.md` 로 한 장씩 쌓는다. **노션으로 그대로 복붙**할 수 있게
깔끔한 마크다운으로 쓴다.

## 절대 원칙

- 여기 적히는 건 **학습자가 스스로 깨달은 것**뿐이다. Claude가 설명해 준 내용은 적지 않는다.
- 학습자가 말한 문장은 **변형하지 않고 원문 그대로 인용**한다. (요약·윤문 금지)
- Claude는 맥락(어떤 질문/시나리오에서 나온 깨달음인지)만 최소한으로 붙인다.
- 원문 인용은 그대로 두고, 그 아래에 **다듬은 정리**(나중에 봐도 이해되게 정돈한 문장)와
  **이해도 점검 / 더 나아가려면**(Claude 평가)을 별도 줄로 덧붙인다. 이모지 대신 굵은 글씨 라벨.
- 이해도 점검은 **이미 완료된 시나리오에만** 단다 (아직 탐구 중이면 정답을 흘리지 않는다).

## 한 장의 형식 (템플릿)

```markdown
# YYYY-MM-DD 깨달음 일지

- 다룬 단계: vN
- 다룬 시나리오: (예: 서버에러 / 렌더에러)

## 오늘 내가 스스로 깨달은 것

### 💡 (깨달음 제목)
- 이끈 질문: "(Claude가 던졌던 질문 원문)"
- 본 코드 (발췌):
  ```js
  // 깨달음을 끌어낸 코드/예제에서 핵심 줄만 짧게 (3~6줄)
  ```
- 내 말 (원문 그대로):
  > (학습자가 실제로 한 말 그대로)
- **다듬은 정리**: (원문을 나중에 봐도 이해되게 정돈한 문장 — 원문은 위에 그대로 보존)
- 맥락: (어느 코드/시나리오에서 나왔는지 한 줄)
- **이해도 점검**: (정확히 짚은 부분 / 더 정밀하게 다듬을 부분 — 완료된 시나리오만)
- **더 나아가려면**: (한 걸음 더 갈 다음 질문 한 가지)

## 아직 안 풀린 / 다음에 이어갈 질문
- (학습자가 끝까지 못 답했거나 다음으로 미룬 것 — 원문 보존)
```
