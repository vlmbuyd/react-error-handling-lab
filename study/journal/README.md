# 학습 일지 (journal)

`/wrap`이 세션 끝에 `YYYY-MM-DD.md` 로 한 장씩 쌓는다. **노션으로 그대로 복붙**할 수 있게
깔끔한 마크다운으로 쓴다.

- 파일 제목(`# ...`)은 **그날 학습한 내용을 짧게 요약한 주제**로 짓는다.
  (예: `# 2026-06-15 — try/catch 런타임 에러 시나리오 실습`) — "깨달음 일지" 같은 고정 문구 금지.

## 절대 원칙

- 여기 적히는 건 **학습자가 스스로 깨달은 것**뿐이다. Claude가 설명해 준 내용은 적지 않는다.
- 학습자가 말한 문장은 **변형하지 않고 원문 그대로 인용**한다. (요약·윤문 금지)
- Claude는 맥락(어떤 질문/시나리오에서 나온 깨달음인지)만 최소한으로 붙인다.
- 원문 인용은 그대로 두고, 그 아래에 **다듬은 정리**와 **점검 / 다음 한 걸음**을 별도 줄로 덧붙인다.
- 점검은 **이미 완료된 시나리오에만** 단다 (아직 탐구 중이면 정답을 흘리지 않는다).

## 쓰기 스타일 (노션 복붙용)

- **짧게.** 한 문장에 한 가지. 정리 문장은 2~3줄 이내.
- **표준 용어만.** 이 저장소·대화에서만 통하는 은어("닫는 질문", "capstone" 등)는 정리·요약에 쓰지 않는다.
- 학습자가 만든 비유는 **원문 인용에만** 남기고, 정리에서는 풀어 쓴다.
- 중첩 인용·과한 들여쓰기·이모지 라벨 금지. **굵은 글씨 라벨**로 구분.
- 맨 아래에 **한눈에 복습** 섹션을 둔다 (핵심을 항목으로만, 혼자 정리용).

## 한 장의 형식 (템플릿)

```markdown
# YYYY-MM-DD — (그날 학습 내용 요약 제목)

- 다룬 단계: vN
- 다룬 시나리오: (예: 서버에러 / 렌더에러)

## 오늘 내가 스스로 깨달은 것

### (깨달음 제목)
- 이끈 질문: "(Claude가 던졌던 질문 원문)"
- 본 코드:
  ```js
  // 핵심 3~6줄만
  ```
- 내 말 (원문 그대로):
  > (학습자가 실제로 한 말 그대로)
- **다듬은 정리**: (2~3줄, 표준 용어 — 원문은 위에 그대로 보존)
- 맥락: (어느 코드/시나리오에서 나왔는지 한 줄)
- **점검**: (정확히 짚은 부분 / 더 정밀히 할 부분 — 완료된 시나리오만)
- **다음 한 걸음**: (한 발 더 갈 질문 하나)

## 한눈에 복습 (내 말로 다시 설명해보기)
- **(핵심 항목 1)** — (떠올릴 단서 한 줄)
- **(핵심 항목 2)** — (떠올릴 단서 한 줄)

## 아직 안 풀린 / 다음에 이어갈 질문
- (학습자가 끝까지 못 답했거나 다음으로 미룬 것 — 원문 보존)
```
