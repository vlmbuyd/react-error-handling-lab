# /wrap — 세션 마무리 & 깨달음 일지 기록

이번 세션의 대화를 되돌아보고 **학습자가 스스로 깨달은 것**만 추려 일지로 남긴다.
이 일지는 **노션으로 그대로 복붙**되므로 깔끔한 마크다운으로 쓴다.

## 절대 원칙 (원문 보존)

- 기록 대상은 **학습자(사람)가 스스로 도달해 말한 통찰**뿐이다.
  **Claude가 설명/유도한 내용은 적지 않는다.** 네 말이 아니라 학습자의 말이 자산이다.
- 학습자가 한 문장은 **변형·요약·윤문 없이 원문 그대로 인용**한다. (`>` 인용 블록)
- Claude는 *맥락*(어떤 질문/시나리오/코드에서 나온 깨달음인지)만 한 줄로 최소한 덧붙인다.
- 확신이 안 서면 지어내지 말고, 학습자에게 "오늘 가장 크게 깨달은 한 가지를 네 말로 적어줄래?"라고 물어 그대로 받는다.

## 다듬은 정리 + 이해도 점검 (원문 보존 블록과 분리해서 추가)

원문 인용(`>` 블록)은 **절대 손대지 않고 그대로 둔다.** 그 아래에 별도 줄로 다음을 덧붙인다.
**이모지는 쓰지 않고 굵은 글씨 텍스트 라벨**로 구분한다.

- **다듬은 정리**: 학습자가 말한 원문을, 나중에 다시 봤을 때 한눈에 이해되도록 정돈된 문장으로 정리한다.
  원문을 대체하는 게 아니라 **원문은 그대로 두고 그 아래에 정돈본을 추가**하는 것이다.
  내용(주장·인과)은 학습자의 것을 그대로 살리되 표현만 또렷하게 다듬는다 — 새로운 통찰을 끼워넣지 않는다.
- **이해도 점검**: 학습자가 정확히 짚은 부분과, 부정확하거나 더 정밀하게 다듬을 부분을 솔직하게 적는다.
  틀린 게 없으면 "정확함"이라고 적고, 어설픈 표현(부정확한 용어·인과)이 있으면 콕 집어 교정한다.
- **더 나아가려면**: 이 깨달음에서 한 걸음 더 갈 다음 질문/확인 한 가지.

**단, 소크라테스 원칙은 유효하다 — 아직 탐구 중(🟡)이거나 안 푼(🔴) 깨달음에는
정답을 미리 흘리는 점검을 적지 않는다.** 이해도 점검은 **이미 클리어(완료)된 시나리오**에만 단다.
아직 열린 질문이면 점검 줄을 비우고 "다음 세션에서 점검"이라고만 남긴다.

## 절차

1. 이번 세션 대화를 훑어 **학습자가 스스로 깨달음을 말한 순간**들을 찾는다.
   (Claude 질문에 학습자가 "아 그래서~", "그러니까 ~인 거네" 식으로 자기 말로 답한 지점)
2. `study/journal/README.md`의 템플릿 형식으로 **오늘 날짜** 파일 `study/journal/YYYY-MM-DD.md`를 쓴다.
   같은 날 파일이 이미 있으면 **이어붙인다**(덮어쓰지 않는다).
3. `study/PROGRESS.md`를 갱신한다:
   - 현재 버전 / 마지막 세션 날짜 / 다음 세션 시작점
   - 핵심 질문 3개 상태 (🔴🟡🟢) — 학습자가 자기 말로 설명 가능해졌으면 🟢
   - 버전별 체크포인트 체크
   - "아직 열려있는 질문" — 학습자가 끝까지 못 답했거나 미룬 것(원문 보존)
4. 마지막에 학습자에게 **무엇을 기록했는지 짧게 보여주고**, 빠진 게 없는지 확인받는다.

## 일지 형식 (study/journal/README.md 참고)

```markdown
# YYYY-MM-DD 깨달음 일지

- 다룬 단계: vN
- 다룬 시나리오: ...

## 오늘 내가 스스로 깨달은 것

### 💡 (깨달음 제목)
- 이끈 질문: "(Claude가 던졌던 질문 원문)"
- 내 말 (원문 그대로):
  > (학습자가 실제로 한 말 그대로)
- **다듬은 정리**: (원문을 나중에 봐도 이해되게 정돈한 문장 — 원문은 위에 그대로 보존)
- 맥락: (어느 코드/시나리오에서 나왔는지 한 줄)
- **이해도 점검**: (정확히 짚은 부분 / 부정확하거나 더 정밀하게 다듬을 부분 — 완료된 시나리오만)
- **더 나아가려면**: (한 걸음 더 갈 다음 질문 한 가지)

## 아직 안 풀린 / 다음에 이어갈 질문
- (원문 보존)
```

## 하지 말 것

- 학습자가 말하지 않은 깨달음을 Claude가 지어내거나 채워 넣기.
- **원문 인용(`>` 블록)을 다듬어 덮어쓰기** → 정리본은 따로, 원문은 그대로 둔다.
- "다듬은 정리"에 학습자가 말 안 한 새 통찰·결론 끼워넣기 → 표현만 또렷하게, 내용은 학습자 것.
- Claude가 한 설명을 학습자의 깨달음인 양 적기.
- 이모지 라벨 사용 → **굵은 글씨 텍스트 라벨**로 구분.
- **아직 안 푼/탐구 중인** 깨달음에 정답을 흘리는 점검 달기 → 소크라테스 원칙 위반.
