# /wrap — 세션 마무리 & 깨달음 일지 기록

이번 세션에서 **학습자가 스스로 깨달은 것**만 추려 일지로 남긴다.
일지는 **노션에 그대로 복붙**하므로, 표준 마크다운으로 짧고 읽기 쉽게 쓴다.

## 절대 원칙 (원문 보존)

- 기록 대상은 **학습자가 스스로 도달해 말한 것**뿐이다. Claude가 설명·유도한 내용은 적지 않는다.
- 학습자가 한 말은 **변형·요약·윤문 없이 그대로 인용**한다. (`>` 인용 블록)
- Claude는 *맥락*(어느 코드·시나리오에서 나온 말인지)만 한 줄로 덧붙인다.
- 깨달음을 끌어낸 **코드는 핵심 3~6줄만 발췌**한다. 전체 붙여넣기 금지.
- 확신이 안 서면 지어내지 말고, "오늘 가장 크게 깨달은 한 가지를 네 말로 적어줄래?"라고 물어 그대로 받는다.

## 쓰기 스타일 (노션 복붙용 — 반드시 지킬 것)

- **짧게 쓴다.** 한 문장에 한 가지만. 길게 늘어놓지 말고 사고의 흐름이 드러나게.
- **표준 용어만 쓴다.** 이 저장소·대화에서만 통하는 표현(예: "닫는 질문", "닫았다",
  "capstone", "consolidation")을 정리·요약 문장에 쓰지 않는다. 누구나 아는 말로 바꾼다.
- 학습자가 직접 만든 비유는 **원문 인용에만** 남기고, 정리 문장에서는 풀어 쓴다.
- 중첩 인용·과한 들여쓰기·이모지 라벨을 쓰지 않는다. **굵은 글씨 라벨**로 구분한다.
- 항목 하나당 정리 문장은 **2~3줄**을 넘기지 않는다.

## 정리 + 점검 (원문 아래에 별도로)

원문 인용(`>` 블록)은 **손대지 않는다.** 그 아래에 다음을 짧게 덧붙인다.

- **다듬은 정리**: 원문을 나중에 봐도 이해되게 다듬은 2~3줄. 내용은 학습자 것 그대로,
  표현만 또렷하게. 새 통찰을 끼워넣지 않는다.
- **점검**: 정확히 짚은 부분과 더 정밀히 할 부분을 솔직하게. 틀린 게 없으면 "정확함".
- **다음 한 걸음**: 여기서 한 발 더 갈 질문 하나.

**소크라테스 원칙은 유효하다.** 점검은 **이미 끝난(완료된) 시나리오**에만 단다.
아직 탐구 중(🟡)·미해결(🔴)이면 점검을 비우고 "다음 세션에서 점검"이라고만 적는다.

## 마지막 섹션 — 한눈에 복습 (혼자 정리용)

일지 맨 아래에 **이번 세션 핵심을 항목으로만** 추린 복습 목록을 둔다.
학습자가 이걸 보고 **자기 말로** 다시 설명하며 자기 것으로 만들기 위한 칸이다.

- 각 항목은 **핵심 한 줄(소제목)** + 떠올릴 단서 한 줄. 길게 설명하지 않는다.
- 끝난 시나리오는 결론을 한 줄로 적어도 되지만, **자세한 풀이는 적지 않는다**(스스로 채우게).
- 아직 탐구 중인 항목은 **질문 형태로만** 남긴다(정답 흘리기 금지).

## 절차

1. 이번 세션 대화에서 **학습자가 자기 말로 깨달음을 말한 순간**을 찾는다.
   (Claude 질문에 "아 그래서~", "그러니까 ~인 거네" 식으로 답한 지점)
2. **오늘 날짜** 파일 `study/journal/YYYY-MM-DD.md`를 템플릿 형식으로 쓴다.
   같은 날 파일이 있으면 **이어붙인다**(덮어쓰지 않는다).
3. `study/PROGRESS.md`를 갱신한다:
   - 현재 버전 / 마지막 세션 날짜 / 다음 시작점
   - 핵심 질문 3개 상태(🔴🟡🟢) — 자기 말로 설명 가능하면 🟢
   - 버전별 체크포인트 / "아직 열린 질문"(원문 보존)
4. 학습자에게 **무엇을 기록했는지 짧게 보여주고** 빠진 게 없는지 확인받는다.

## 일지 형식 (study/journal/README.md 참고)

```markdown
# YYYY-MM-DD — (그날 학습 내용 요약 제목)

- 다룬 단계: vN
- 다룬 시나리오: ...

## 오늘 내가 스스로 깨달은 것

### (깨달음 제목)
- 이끈 질문: "(Claude가 던졌던 질문 원문)"
- 본 코드:
  ```js
  // 핵심 3~6줄만
  ```
- 내 말 (원문 그대로):
  > (학습자가 실제로 한 말)
- **다듬은 정리**: (2~3줄, 표준 용어)
- 맥락: (어느 코드·시나리오에서 나왔는지 한 줄)
- **점검**: (정확히 짚은 부분 / 더 정밀히 할 부분 — 완료된 시나리오만)
- **다음 한 걸음**: (한 발 더 갈 질문 하나)

## 한눈에 복습 (내 말로 다시 설명해보기)
- **(핵심 항목 1)** — (떠올릴 단서 한 줄)
- **(핵심 항목 2)** — (떠올릴 단서 한 줄)

## 아직 안 풀린 / 다음에 이어갈 질문
- (원문 보존)
```

## 하지 말 것

- 학습자가 말하지 않은 깨달음을 지어내거나 채워 넣기.
- **원문 인용(`>` 블록)을 다듬어 덮어쓰기** → 정리는 따로, 원문은 그대로.
- "다듬은 정리"에 학습자가 말 안 한 새 통찰 끼워넣기 → 표현만 또렷하게.
- 저장소·대화에서만 통하는 은어를 정리·요약에 쓰기 → 표준 용어로.
- 이모지 라벨 사용 → **굵은 글씨 라벨**.
- 길고 늘어지는 문장 → 한 문장에 한 가지, 2~3줄 이내.
- **탐구 중·미해결** 항목에 정답 흘리기 → 소크라테스 원칙 위반.
